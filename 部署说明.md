## B. 你需要手动完成的步骤（约 10 分钟）

### 1）把代码放到 GitHub

本地已完成首次 commit（`886c509`）。还需登录 GitHub 后推送：

```powershell
cd C:\Users\36128\paper-trading
# 安装过 GitHub CLI 后，在终端登录（会打开浏览器）
& "C:\Program Files\GitHub CLI\gh.exe" auth login --web --git-protocol https
# 创建仓库并推送（二选一）
& "C:\Program Files\GitHub CLI\gh.exe" repo create paper-trading --public --source=. --remote=origin --push
# 或手动：先在 https://github.com/new 建空仓库，再：
# git remote add origin https://github.com/<你的用户名>/paper-trading.git
# git branch -M main
# git push -u origin main
```

### 2）在 Render 部署

1. 打开 https://dashboard.render.com 注册/登录（可用 GitHub 登录）
2. **New** → **Web Service**
3. 连接刚才的 `paper-trading` 仓库
4. 填写：
   - **Name**: `paper-trading`
   - **Language**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python -m uvicorn app.main:app --host 0.0.0.0 --port $PORT`
5. 计划可选 Free
6. 点击 **Create Web Service**，等待 Deploy 成功
7. 打开 Render 给的网址，例如 `https://paper-trading-xxxx.onrender.com`

也可：**New** → **Blueprint** → 选择仓库，自动读取 `render.yaml`。

### 3）验证

- 打开首页应看到欢迎页（不是 Page not found）
- 访问 `/api/health` 应返回 `{"ok": true, ...}`

---

## C. 注意事项

- Free 实例一段时间无访问会休眠，首次打开可能要等 30–60 秒
- SQLite 在免费实例重启后可能丢数据；学习演示可用，正式长期数据建议以后换 Postgres
- 国内访问 Render 域名偶发较慢，属网络问题
